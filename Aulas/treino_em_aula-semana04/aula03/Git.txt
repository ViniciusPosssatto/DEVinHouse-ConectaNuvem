*commit = pacote de alterações (quando vai anexar uma mudança no código, vamos comitar algo, atribuindo um  comentário sobre o que foi feito/o que foi modificado);

*branch = ramos, divisões, 'galhos' de uma árvore. Baixar uma versão da branch principal e ter uma branch 'cópia' para fazer as modificações e depois mesclar na branch principal.

*merge = mesclar, juntar uma branch com a outra. Não é necessário mesclar tudo, pode ser selecionado para mesclar apenas os comits que não estão nessa outra branch (as vezes você baixa uma branch que possui outros comits em comum com a sua).

*pull request = antes de gerar as alterações para a branch principal, ela fica em espera até alguém testar essa brach nova, para averiguar se está tudo funcionando certo, para ai sim, anexar à branch pricipal (para que não ocorra de entrar direto na principal e causar problemas).

*branch-local = meu pc.
*branch=remote/origin = no servidor.

*push = enviar as alterações locais que não estão no remote.

*fetch = verifica os arquivos no remote, para ver se possui alguma alteração que não está no local e poder assim baixá-las para ficarem iguais(commits, versões, etc.).

*pull = puxar as alterações que estão no remote para o local (se caso houver alguma que não está no local).
